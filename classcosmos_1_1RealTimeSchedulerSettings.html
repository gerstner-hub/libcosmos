Detailed Description

Base class for realtime scheduling policies.

Realtime scheduling uses priorities between an integer min priority and an integer max priority. These boundaries are determines during runtime but are currently set on Linux to 1 .. 99. Higher values mean hihger priorities.

A thread with realtime scheduling always has a higher priority than threads with non-realtime scheduling.
